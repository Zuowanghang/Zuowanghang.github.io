MR

团队成员通过创建分支进行写作，成员做出更改后将代码推送到自己的分支上，新建从自己的分支到主干仓库的合并请求。

回合

主干仓库接收多个成员的贡献，版本领先于其他分支。成员需要将主干仓库中的代码更新到自己的分支。
